## 1. User Domain

### 책임

- 시스템 접근 주체
- 모든 행위의 최상위 책임 단위

### 포함 개념

- 계정
- 기본 사용자 정보
- 차단 / 신고의 주체

### 제외

- 실명 인증
- 결제 정보
- 외부 연동 정보

---

## 2. Dog Domain

### 책임

- 반려견에 대한 **정보 표현 단위**
- 만남에서의 주요 판단 기준 제공

### 포함 개념

- 반려견 기본 정보
- 반려견 성향
- 반려견 주의사항

### 관계

- 하나의 User는 여러 Dog를 가질 수 있다
- Dog는 독립적으로 행위를 하지 않는다

---

## 3. Survey Domain (Dog Profile Detail)

### 책임

- 반려견 단위의 성향/특성 데이터 수집

### 포함 개념

- 성격 질문
- 주의사항 응답
- 선택형/서술형 응답

### 관계

- Survey는 Dog에 종속된다
- User에는 직접 연결되지 않는다

---

## 4. Meetup Domain

### 책임

- 오프라인 만남을 표현하는 **중심 도메인**

### 포함 개념

- 모임 생성
- 시간 / 장소 / 조건
- 최대 참여 인원

### 제외

- 자동 매칭
- 추천 로직

---

## 5. Participation Domain

### 책임

- 모임과 사용자 간의 **참여 상태 관리**

### 포함 개념

- 참여 요청
- 승인 / 거절 상태
- 참여 이력

### 관계

- User ↔ Meetup 연결 엔티티
- 상태를 가진다 (단순 연결 아님)

---

## 6. Communication Domain (Restricted)

### 책임

- 모임 단위의 제한된 커뮤니케이션

### 포함 개념

- 모임 공지
- 모임 관련 메시지

### 제외

- 1:1 메시지
- 외부 연락 수단 교환

---

## 7. Safety Domain

### 책임

- 최소한의 운영 안전 장치 제공

### 포함 개념

- 신고
- 차단

### 관계

- User를 기준으로 작동
- Meetup, Communication과 연관 가능

---

## 8. 도메인 간 관계 요약 (개념)

```
User
 ├─ Dog
 │    └─ Survey
 │
 ├─ Participation ─ Meetup
 │
 └─ Safety (Report / Block)

```
