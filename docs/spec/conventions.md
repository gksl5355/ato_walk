# conventions.md

## 0. Purpose
본 문서는 본 프로젝트의 **코드 스타일, 표현 방식, 주석 규칙**을 정의한다.  
본 문서는 구조, 기술 선택, 아키텍처 판단을 포함하지 않는다.

---

## 1. 기본 원칙

- 가독성을 최우선으로 한다
- 일관성을 개별 취향보다 우선한다
- `templates/**`와 본 문서가 충돌하면 **`templates/**`를 따른다**
- 단, `templates/**`는 `docs/spec/spec.md` 우선순위를 위반할 수 없다
- 템플릿이 부족하면 개별 구현에서 벗어나지 말고 템플릿을 먼저 수정한다

---

## 2. Java 스타일

- Google Java Style Guide를 기본으로 따른다
- Formatter/Checkstyle 도입은 필수가 아니다
- 과도한 스타일 강제는 지양한다

---

## 3. 네이밍 규칙

### 클래스
- PascalCase
- 역할이 드러나는 명사

### 메서드
- camelCase
- 동사 + 목적어 형태

### 패키지
- 소문자
- 도메인 단위 명사

---

## 4. 주석 규칙

### 주석을 작성하는 경우
- 비즈니스 규칙의 **의도**가 드러나지 않을 때
- 도메인 판단 근거가 필요할 때

### 작성하지 말아야 할 주석
- 코드만 봐도 명확한 내용
- 단순 동작 설명

---

## 5. Controller 규칙

- 요청/응답 처리만 담당
- 비즈니스 로직 금지
- Repository 직접 호출 금지

---

## 6. Service 규칙

- 하나의 메서드는 하나의 유스케이스
- 트랜잭션 경계는 Service에만 둔다
- 상태 전이는 Entity 메서드를 통해 수행한다

---

## 7. Entity 규칙

- setter 남용 금지
- 의미 없는 public 필드 금지
- 상태 변경은 의도를 드러내는 메서드로 표현

---

## 8. Exception 규칙

- RuntimeException 기반
- 도메인 의미가 드러나는 예외만 정의
- 메시지는 디버깅 기준으로 작성

---

## 9. Logging 규칙

- 비즈니스 흐름의 핵심 지점만 로그
- 개인정보 로그 금지
- 디버그 목적 로그 남발 금지
